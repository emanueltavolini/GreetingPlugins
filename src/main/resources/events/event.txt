English:
Events:
In the event1.yml file, you can create different events for when a player joins the server.
The available event types (type) are:
1) command: A specified command will be executed automatically when a player joins the server:
events:
  - type: command --> Defines the type of event.
    command: say Command --> command to execute.
    delay: 40 --> Delay in ticks, to execute the command.
    once: false --> If true the command will only be executed when the player enters the server for the first time.
    enable: false --> If it is true the event will be active, but when it turns false the event is deactivated.

2) message: A message will be sent to the player when they enter the server:
events:
  - type: message --> Defines the type of event.
    message: Message --> Message to send to the player
    delay: 40 --> Delay in ticks, to send the message.
    once: false --> If true the message will only be sent when the player enters the server for the first time.
    enable: false --> If it is true the event will be active, but when it turns false the event is deactivated.

3) give: An item will be given to the player when they enter:
events:
  - type: give --> Defines the type of event.
    item: grass_block --> Item to give to the player.
    amount: 1 --> Quantity of the item to be delivered.
    delay: 40 --> Delay in ticks, to send the message.
    once: true --> If true the item will only be given when the player enters the server for the first time.
    enable: false --> If it is true the event will be active, but when it turns false the event is deactivated.

Structure:
events:
  - type: message
    message: Type message
    delay: 40
    once: false
    enable: false
  - type: command
    command: say type command
    delay: 40
    once: false
    enable: false
  - type: give
    item: grass_block
    amount: 1
    delay: 40
    once: false
    enable: false

Deutsch:
Veranstaltungen (events):
In der Datei event1.yml können Sie verschiedene Ereignisse für den Beitritt eines Spielers zum Server erstellen.
Die verfügbaren Ereignistypen (type) sind:

1) command: Ein festgelegter Befehl wird automatisch ausgeführt, wenn ein Hacker den Server betritt:
events:
  - type: command --> Definiert den Ereignistyp.
    command: say Befehl --> Befehl zur Ausführung.
    delay: 40 --> Verzögerung in Ticks, um den Befehl auszuführen.
    once: false --> Wenn „true“, wird der Befehl nur ausgeführt, wenn der Spieler den Server zum ersten Mal betritt.
    enable: false --> Wenn der Wert „true“ ist, ist das Ereignis aktiv, wenn der Wert jedoch „false“ ist, wird das Ereignis deaktiviert.

2) message: Beim Betreten des Servers wird dem Spieler eine Nachricht gesendet:
events:
  - type: message --> Definiert den Ereignistyp.
    message: Nachricht --> An den Spieler zu sendende Nachricht.
    delay: 40 --> Verzögerung in Ticks, um die Nachricht zu senden.
    once: false --> Wenn „true“, wird die Nachricht nur gesendet, wenn der Spieler den Server zum ersten Mal betritt.
    enable: false --> Wenn der Wert „true“ ist, ist das Ereignis aktiv, wenn der Wert jedoch „false“ ist, wird das Ereignis deaktiviert.

3) give: se le dara un item al jugador cuando entré:
events:
  - type: give --> Definiert den Ereignistyp.
    item: grass_block --> Gegenstand, der dem Spieler gegeben werden soll.
    amount: 1 --> Menge des zu liefernden Artikels.
    delay: 40 --> Verzögerung in Ticks, um dem Spieler den Gegenstand zu geben.
    once: true --> Wenn dies der Fall ist, wird der Gegenstand nur ausgegeben, wenn der Spieler den Server zum ersten Mal betritt.
    enable: true --> Wenn der Wert „true“ ist, ist das Ereignis aktiv, wenn der Wert jedoch „false“ ist, wird das Ereignis deaktiviert.

Struktur:
events:
  - type: message
    message: Type message
    delay: 40
    once: false
    enable: false
  - type: command
    command: say type command
    delay: 40
    once: false
    enable: false
  - type: give
    item: grass_block
    amount: 1
    delay: 40
    once: false
    enable: false
